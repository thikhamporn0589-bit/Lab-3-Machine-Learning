# การออกแบบ Training Strategy

## 1. การกำหนดประเภทที่ดิน (Land Cover Classes)

ในการศึกษานี้ได้กำหนดประเภทการใช้ที่ดินทั้งหมด 5 ประเภท ได้แก่

1. Water (0): แหล่งน้ำ เช่น แม่น้ำ คลอง หรือแหล่งน้ำถาวร ซึ่งมีค่าการสะท้อนแสงในช่วง NIR ต่ำมาก
2. Forest (1): พื้นที่ป่าไม้หรือพืชพรรณหนาแน่น มีค่า NDVI สูง
3. Agriculture (2): พื้นที่เกษตรกรรม เช่น นาข้าว หรือพืชไร่ ซึ่งมีค่า NDVI ปานกลางและมีการเปลี่ยนแปลงตามฤดูกาล
4. Urban (3): พื้นที่สิ่งปลูกสร้าง เช่น อาคาร ถนน ซึ่งมีค่าการสะท้อนในช่วง SWIR สูง
5. Bare land (4): พื้นที่ดินเปล่าหรือพื้นที่ที่ไม่มีพืชปกคลุม มีค่า NDVI ต่ำ

การเลือกจำนวน 5 ประเภทนี้ช่วยให้สามารถแยกประเภทที่ดินได้ละเอียดเพียงพอ และยังคงประสิทธิภาพของโมเดลให้อยู่ในระดับที่เหมาะสม

---

## 2. วิธีการสร้าง Training Samples

ในการสร้างข้อมูลฝึก (training samples) ใช้ข้อมูลอ้างอิงจาก Dynamic World ร่วมกับการปรับให้เหมาะสมกับพื้นที่ศึกษา

Dynamic World เป็นชุดข้อมูลที่ให้ label การใช้ที่ดินในระดับโลก แต่ยังมีความคลาดเคลื่อนในบางพื้นที่ ดังนั้นจึงมีการ remap class ให้เหลือ 5 ประเภทตามที่กำหนด

ตัวอย่างข้อมูลถูกสุ่มภายในขอบเขตพื้นที่ศึกษา (ROI) โดยไม่ได้คัดลอกจาก tutorial โดยตรง เพื่อให้ข้อมูลสะท้อนลักษณะพื้นที่จริง

แนวทางนี้ช่วยให้สามารถสร้างข้อมูลฝึกได้อย่างรวดเร็ว มีความสม่ำเสมอ และเหมาะสมกับพื้นที่ศึกษา

---

## 3. การแบ่งข้อมูล Train / Validation

ข้อมูลถูกแบ่งเป็น 2 ส่วน ได้แก่

* Training set 80%
* Validation set 20%

โดยใช้วิธีการสุ่ม (random sampling)

ข้อดีของวิธีนี้คือสามารถใช้งานได้ง่าย และทำให้โมเดลเรียนรู้จากข้อมูลจำนวนมากได้

อย่างไรก็ตาม วิธีนี้อาจเกิดปัญหา spatial autocorrelation เนื่องจากข้อมูลที่อยู่ใกล้กันอาจถูกแบ่งไปอยู่ทั้งในชุด train และ validation ทำให้ค่าความแม่นยำสูงเกินจริง

อีกทางเลือกหนึ่งคือ spatial cross-validation ซึ่งแบ่งข้อมูลตามตำแหน่งเชิงพื้นที่ ทำให้การประเมินผลมีความสมจริงมากขึ้น แต่มีความซับซ้อนในการดำเนินการ

ในการศึกษานี้เลือกใช้ random sampling เนื่องจากมีความเรียบง่ายและเหมาะสมกับเวลาในการทำงาน

---

## 4. การเลือก Features

Features ที่ใช้ในการจำแนก ได้แก่

* แบนด์สเปกตรัม: B2 (Blue), B3 (Green), B4 (Red), B5–B7 (Red Edge), B8 (NIR), B11 (SWIR)
* ดัชนีพืชพรรณ: NDVI
* ดัชนีน้ำ: NDWI

เหตุผลในการเลือก ได้แก่

* NDVI ช่วยแยกพื้นที่ที่มีพืชออกจากพื้นที่ที่ไม่มีพืช
* NDWI ช่วยตรวจจับแหล่งน้ำได้อย่างมีประสิทธิภาพ
* Red Edge bands (B5–B7) ช่วยแยกประเภทของพืชพรรณ เช่น ป่าและเกษตรกรรม
* SWIR (B11) ช่วยแยกพื้นที่สิ่งปลูกสร้างและความชื้นของดิน

การรวมทั้ง spectral bands และ indices ช่วยให้โมเดลสามารถเรียนรู้ลักษณะของพื้นที่ได้ดีขึ้น และเพิ่มความแม่นยำในการจำแนก






# คำถามวิเคราะห์ผลการทดลอง

## 1. หากเพิ่ม Training Samples เป็น 2 เท่า ความแม่นยำเพิ่มขึ้นเท่าใด

จากการทดลองเพิ่มจำนวน training samples จาก 4000 เป็น 8000 พบว่าความแม่นยำ (accuracy) เพิ่มขึ้นเพียงเล็กน้อย โดยอยู่ในช่วงประมาณ 3–5%

สาเหตุที่ความแม่นยำเพิ่มขึ้น เนื่องจากโมเดลได้รับข้อมูลที่หลากหลายมากขึ้น ทำให้สามารถเรียนรู้ลักษณะของแต่ละ class ได้ดีขึ้น โดยเฉพาะในพื้นที่ที่มีความซับซ้อน เช่น พื้นที่เกษตรและพื้นที่ป่า

อย่างไรก็ตาม การเพิ่มข้อมูลไม่ได้ทำให้ความแม่นยำเพิ่มขึ้นอย่างมาก เนื่องจากเกิดปรากฏการณ์ diminishing returns คือ เมื่อข้อมูลมีมากเพียงพอแล้ว การเพิ่มข้อมูลจะช่วยได้เพียงเล็กน้อยเท่านั้น

---

## 2. Spatial Autocorrelation มีผลต่อความแม่นยำอย่างไร

Spatial autocorrelation คือการที่ข้อมูลที่อยู่ใกล้กันในเชิงพื้นที่มีลักษณะคล้ายกัน

ในกรณีที่ใช้การสุ่มแบ่งข้อมูล (random split) พิกเซลที่อยู่ใกล้กันอาจถูกแบ่งไปอยู่ทั้งในชุด training และ validation ทำให้โมเดลสามารถทำนายได้ง่ายขึ้น ส่งผลให้ค่าความแม่นยำที่รายงานสูงเกินความเป็นจริง

ดังนั้น accuracy ที่ได้อาจไม่สะท้อนประสิทธิภาพของโมเดลเมื่อใช้งานในพื้นที่ใหม่

วิธีแก้ปัญหานี้คือการใช้ spatial cross-validation ซึ่งแบ่งข้อมูลตามพื้นที่จริง ทำให้การประเมินผลมีความสมจริงมากขึ้น

---

## 3. Class ที่โมเดลทำได้แย่ที่สุด และแนวทางแก้ไข

จากผลการจำแนก พบว่า class ที่โมเดลทำได้แย่ที่สุดคือ Agriculture และ Forest

สาเหตุหลักเกิดจากทั้งสอง class มีลักษณะสเปกตรัมคล้ายกัน โดยเฉพาะในพื้นที่ที่พืชมีความหนาแน่นสูง ทำให้โมเดลแยกได้ยาก

แนวทางแก้ไข ได้แก่:

* เพิ่ม features เช่น Red Edge bands (B5–B7) เพื่อช่วยแยกประเภทพืช
* เพิ่มจำนวน training samples ให้ครอบคลุมความหลากหลายของข้อมูล
* ใช้ indices เพิ่มเติม เช่น NDVI หรือ indices อื่นที่เกี่ยวกับพืชพรรณ
* ปรับ threshold หรือปรับ label ให้เหมาะสมกับพื้นที่

---

## 4. หากต้องทำซ้ำในพื้นที่อื่น ต้องปรับอะไร และใช้อะไรซ้ำได้

สิ่งที่ต้องปรับ:

* ขอบเขตพื้นที่ศึกษา (ROI)
* นิยาม class ให้เหมาะสมกับพื้นที่ เช่น พื้นที่เมืองหรือพื้นที่ป่า
* training samples เนื่องจากลักษณะพื้นที่แต่ละแห่งแตกต่างกัน

สิ่งที่สามารถใช้ซ้ำได้:

* โครงสร้าง workflow เช่น การใช้ Sentinel-2, Dynamic World และ Random Forest
* ขั้นตอน feature engineering เช่น การใช้ NDVI และ NDWI
* วิธีการ train และ evaluate model

ดังนั้น workflow สามารถนำไปใช้ซ้ำได้ แต่ต้องปรับข้อมูลและ parameter ให้เหมาะสมกับพื้นที่ใหม่
